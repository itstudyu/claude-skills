# Skill Catalog Manager

Scan all installed skill sources, parse SKILL.md frontmatter, and maintain
`skill-catalog.md` as the unified skill registry.

## Commands

| Command | Usage | Description |
|---------|-------|-------------|
| `scan` | `/skill-catalog scan` | Full scan of ~/.claude/skills/ + project local skills |
| `add` | `/skill-catalog add <name> <path>` | Register a new skill pack source |
| `list` | `/skill-catalog list` | Show current catalog summary |
| `remove` | `/skill-catalog remove <source>` | Remove a source section from catalog |

## Pipeline

### Phase 1: Detect Sources

Discover all skill sources:

1. **Global skills directory:** `~/.claude/skills/`
   - Each subdirectory that is a symlink → follow to real path
   - Group skills by their real source (e.g., all gstack skills point to `~/.claude/skills/gstack/`)
   - Standalone skills (not part of a pack) → group as "standalone"

2. **Project local skills:** Scan the current project directory
   - Check standard category directories: `workflow/`, `review/`, `planning/`, `figma/`, `utility/`, etc.
   - Each subdirectory with a SKILL.md is a local skill
   - Source name = project directory basename (e.g., "claude-skills")

3. **Existing catalog:** If `skill-catalog.md` exists, read it to preserve:
   - Manual `[manual]` sections added by the user
   - Custom tags that were manually edited

### Phase 2: Parse Skills

For each discovered SKILL.md:

```bash
# Extract frontmatter fields
name=$(grep "^name:" SKILL.md | head -1 | sed 's/name: *//')
description=$(grep -A5 "^description:" SKILL.md | grep -v "^description:" | grep -v "^---" | head -3 | tr '\n' ' ' | sed 's/  */ /g' | head -c 120)
```

Build a skill entry:
- **Skill:** name from frontmatter
- **Path:** relative or absolute path to the skill directory
- **Description:** first ~120 chars of description (single line)
- **Tags:** extract from description keywords, or use `tags:` field if present

### Phase 3: Generate Catalog

Write `skill-catalog.md` in the project root:

```markdown
# Skill Catalog
<!-- Auto-generated by /skill-catalog scan on YYYY-MM-DD -->
<!-- Manual edits in [manual] sections are preserved across scans -->

## <source-name> (source: <path>)
| Skill | Path | Description | Tags |
|-------|------|-------------|------|
| skill-a | path/to/skill-a/ | First line of description | #tag1 #tag2 |
| skill-b | path/to/skill-b/ | First line of description | #tag3 |

## <another-source> (source: <path>)
| Skill | Path | Description | Tags |
|-------|------|-------------|------|
...
```

**Sorting:** Sources alphabetically, skills alphabetically within each source.

**Tag extraction heuristics:**
- "browser", "browse", "headless" → `#browser`
- "review", "PR", "diff" → `#review`
- "test", "QA", "testing" → `#testing`
- "security", "OWASP", "audit" → `#security`
- "Figma", "design" → `#design`
- "plan", "planning" → `#planning`
- "debug", "investigate", "root cause" → `#debugging`
- "ship", "deploy", "CI/CD" → `#shipping`
- "performance", "benchmark" → `#performance`
- Use the `tags:` frontmatter field if present (takes priority)

### Phase 4: Report

Show a summary:

```
Skill Catalog Updated
═══════════════════════════════
Sources: 2
  gstack (30 skills) — ~/.claude/skills/gstack/
  claude-skills (16 skills) — ./

Total: 46 skills
File: skill-catalog.md
```

## Add Command

`/skill-catalog add <name> <path>`

1. Validate the path exists and contains SKILL.md files
2. Add a new source section to skill-catalog.md
3. Scan and populate the section

## Remove Command

`/skill-catalog remove <source-name>`

1. Remove the matching `## <source-name>` section from skill-catalog.md
2. Report what was removed

## Integration with plan-orchestrator

plan-orchestrator reads `skill-catalog.md` as its Tier 1 source:
- Faster than scanning individual SKILL.md files
- Source attribution enables clear reporting
- Tags enable fast filtering by domain

## Important

- Always preserve `[manual]` sections in the catalog
- Never overwrite user-edited tags
- The catalog is a **cache** — the source of truth is always the SKILL.md files
- Run `/skill-catalog scan` after installing or removing skill packs
